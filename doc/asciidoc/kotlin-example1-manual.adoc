== Conclusion

This example presented:

* How to implement a very simple domain model with just one aggregate: Customer
* How to deploy the domain model into crabzilla-vertx

Crabzilla will run your aggregate root by providing these components:

* https://github.com/crabzilla/crabzilla/blob/master/crabzilla-vertx/src/main/java/io/github/crabzilla/vertx/entity/EntityCommandHttpRpcVerticle.java[EntityCommandHttpRpcVerticle] to receive commands as JSON
* https://github.com/crabzilla/crabzilla/blob/master/crabzilla-vertx/src/main/java/io/github/crabzilla/vertx/entity/EntityCommandHandlerVerticle.java[EntityCommandHandlerVerticle] to validate and process commands against the latest snapshot found for the aggregate
* https://github.com/crabzilla/crabzilla/blob/master/crabzilla-vertx/src/main/java/io/github/crabzilla/vertx/entity/EntityUnitOfWorkRepository.java[EntityUnitOfWorkRepository] to store resulting units of work into a single table, being events and command as JSON
* https://github.com/crabzilla/crabzilla/blob/master/crabzilla-vertx/src/main/java/io/github/crabzilla/vertx/projection/EventsProjectionVerticle.java[EventsProjectionVerticle] to propagate and project events to read model database
